## 🔍 二、版本差异详细分析

### 1. **原版(v1-含标点)** - 198,437字符

**特点**：
- ❌ 包含大量噪声（标点22个，数字1,117个，字母6,969个）
- ❌ 汉字占比仅72.6%，质量较低
- ❌ 未经过任何过滤，直接从训练集n-gram提取
- 🔴 **实验结果**：未直接测试（太大）

**样本**：`, 。 果 枣 1 0 . 长 形 2`

---

### 2. **过滤版(PMI过滤)** - 18,678字符

**特点**：
- ✅ 使用PMI（点互信息）过滤，去除低质量n-gram
- ✅ 已过滤掉标点和数字
- ✅ 汉字占比提升至83.4%
- ⚠️ 但仍包含带标点的词（如"。果"、"形,"、"色,"）
- 🔴 **实验结果**：95.47% F1（低于Baseline 95.51%）

**样本**：`枣吊 结果 果实 。果 枣股 形, 色, 横径 纵径 树体`

**问题**：
```
"。果" ← 句号+果
"形," ← 形+逗号
"色," ← 色+逗号
```
这些都是边界切分错误导致的噪声！

---

### 3. **v2版本(去标点)** - 10,959字符

**特点**：
- ✅ 在过滤版基础上进一步去除带标点的词
- ✅ 汉字占比高达96.4%，质量最高
- ✅ 移除了7,719个噪声字符（41.3%）
- 🔴 **实验结果**：95.46% F1（几乎无改善！）

**样本**：`枣吊 结果 果实 枣股 横径 纵径 树体 中等 叶片 枣头`

**移除的噪声样本**：
```
大树, 、制干品 数4- 宿存。 、运 ,梗洼窄 :分布
枝力弱, 较轻, o(小 枣)17 德、 高,于 ,提高
棱。 果点大, 致密, 重16. 直径0. 针形。
```

---

### 4. **平衡版(Balanced)** - 52,487字符

**特点**：
- ❓ 规模最大（52k字符）
- ✅ 无标点、无数字
- ✅ 汉字占比80.0%
- ⚠️ 但包含一些低质量词（如"。果"、"形,"、"色,"）
- 🔄 **实验结果**：Epoch 12/30，Train F1=91.84%（进行中）

**样本**：`。果 形, 色, 枣吊 结果 果实 中等 枣股 大, 平均`

**疑问**：为什么还包含"。果"、"形,"这类噪声？

| 版本 | 词典规模 | 测试F1 | 状态 | 与Baseline对比 |
|------|---------|--------|------|---------------|
| Baseline（无词典） | - | **95.51%** | ✅ | - |

### 关键发现 ⚠️

1. **去标点几乎无效**：
   ```
   过滤版: 95.47%
   v2版本: 95.46% (仅-0.01%)
   ```
   说明问题不是标点噪声！

2. **SoftLexicon整体无效**：
   ```
   所有版本都低于Baseline (95.51%)
   ```

3. **根本原因推测**：
   - ❌ n-gram匹配太粗糙
   - ❌ 预训练词向量不适合RedJujube领域
   - ❌ 4通道BMES特征表达力不足

## 🔍 四、版本间差异分析

### 过滤版 → v2版本

**移除了7,719个字符（41.3%）**

**移除内容主要是**：
1. **带标点的边界错误词**：
   ```
   大树,  、制干品  宿存。  、运  ,梗洼窄  :分布
   枝力弱,  较轻,  ,提高  棱。  果点大,  致密,
   ```

2. **特殊字符混入**：
   ```
   数4-  o(小  枣)17  德、  高,于  重16.  直径0.
   ```

3. **不完整的词组**：
   ```
   针形。  梗洼窄  枝力弱  果点大
   ```

**但性能几乎不变**：
```
95.47% → 95.46% (仅-0.01%)
```

**结论**：这些噪声并不是性能低下的主要原因！

## 💡 五、结论与建议

### 结论 🎯

1. **SoftLexicon在RedJujube上完全无效**
   - 所有版本（18k、11k、52k）都低于Baseline
   - 去标点、PMI过滤等优化措施均无明显效果

2. **词典质量不是主要问题**
   - v2已经非常干净（96.4%汉字占比）
   - 但性能依然低于Baseline

3. **方法本身可能不适合RedJujube**
   - n-gram提取 + 预训练词向量 + 4通道BMES
   - 这种模式在领域专业性强的数据集上失效

### 建议 ✅

1. **放弃SoftLexicon**
   - 已验证多个版本均无效
   - 继续优化词典质量意义不大

2. **专注ExpertDict**
   - ExpertDict(自动): **96.99%** 🏆
   - 单一特征即超过所有融合方案

3. **等待Balanced结果**
   - 如果52k词典仍无效，彻底确认SoftLexicon无用
   - 可作为论文中的对比实验
