## 简介
`eznlp` 是一个基于 PyTorch 的自然语言处理工具包，支持文本分类、命名实体识别、关系抽取等多种任务。本指南将详细介绍如何在不同操作系统环境下正确安装和配置 eznlp，包括通过 pip 安装依赖项、配置 Python 环境、安装 PyTorch 及 transformers 库的具体命令，并提供常见安装问题的解决方案。

## 环境配置
### Python版本要求
根据项目配置文件 `pyproject.toml`，eznlp 要求 Python 版本不低于 3.8。

### 虚拟环境创建
建议使用 conda 创建独立的虚拟环境以避免依赖冲突：

```bash
$ conda create --name eznlp python=3.11
$ conda activate eznlp
```

此命令创建了一个名为 `eznlp` 的新环境，并安装了 Python 3.11 版本。用户可以根据需要调整 Python 版本，但必须满足最低 3.8 的要求。

### 安装PyTorch
由于 PyTorch 有 CPU 和 GPU 两个版本，建议根据硬件情况选择合适的安装方式。对于支持 CUDA 的系统：

```bash
# CUDA 11.8
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118

# CUDA 12.1
pip install torch torchvision torchaudio
```

对于仅使用 CPU 的系统：

```bash
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
```

### 安装eznlp包
有两种安装方式可供选择：

**作为普通包安装：**
```bash
pip install eznlp
```

**开发模式安装（推荐用于开发贡献）：**
```bash
pip install -e .
```

开发模式安装会将当前目录链接到 Python 环境中，便于修改代码后立即测试。

## 使用Taskfile简化流程
### Taskfile简介
`Taskfile.yml` 提供了一组预定义的任务，可以简化常见的训练和监控操作。使用前需要先安装 `task` 工具：

```bash
# Linux/macOS
brew install go-task/tap/go-task

# 或者使用pipx
pipx install taskipy
```

### 常用任务
根据 `Taskfile.yml` 文件，可用的任务包括：

- `train:msra:softlexicon`: 在 MSRA 数据集上训练 SoftLexicon 模型
- `train:msra:bert-base`: 训练基础 BERT 模型
- `collect:msra`: 收集 MSRA 数据集上的实验结果
- `monitor:gpu`: 监视 GPU 使用情况
- `monitor:gpu:detail`: 详细监视 GPU 使用情况

执行示例：
```bash
task train:msra:softlexicon
```

这些任务封装了复杂的命令行参数，使用户能够更方便地复现实验结果。

## Docker环境配置
### 推荐的Docker镜像
项目推荐使用以下 Docker 镜像进行环境配置：

```bash
docker run --rm -it --gpus=all \
  --mount type=bind,source=${PWD},target=/workspace/eznlp \
  --workdir /workspace/eznlp \
  pytorch/pytorch:2.6.0-cuda12.6-cudnn9-devel
```

该命令创建了一个包含最新 PyTorch 和 CUDA 支持的容器环境，并将当前目录挂载到容器内的 `/workspace/eznlp` 路径。

### Docker配置说明
- `--gpus=all`: 允许容器访问所有 GPU 设备
- `--mount`: 将主机当前目录挂载到容器内，实现文件共享
- `--workdir`: 设置容器内的工作目录
- 使用的镜像是 PyTorch 官方提供的开发版，包含了完整的 CUDA 和 cuDNN 支持

这种配置方式确保了环境的一致性，特别适合在不同机器间迁移或团队协作开发。

## 常见问题排查
### CUDA版本不兼容
当出现 CUDA 相关错误时，首先检查 PyTorch 是否正确识别了 GPU：

```python
import torch
print(torch.cuda.is_available())
print(torch.cuda.device_count())
print(torch.cuda.get_device_name(0))
```

如果 `cuda.is_available()` 返回 `False`，可能是以下原因：
1. 没有正确安装支持 CUDA 的 PyTorch 版本
2. 系统缺少必要的 NVIDIA 驱动
3. CUDA 版本与 PyTorch 不匹配

解决方案是重新安装与系统 CUDA 版本匹配的 PyTorch。

### jieba分词器缺失
虽然 jieba 已列在依赖列表中，但在某些环境下可能安装失败。可单独安装：

```bash
pip install jieba
```

验证安装：
```python
import jieba
print("jieba installed successfully")
```

### 预训练模型下载失败
项目使用 `load_pretrained` 函数加载预训练模型（如 BERT）。若下载失败，可能的原因包括：
1. 网络连接问题
2. Hugging Face 服务器访问受限

解决方案：
- 使用国内镜像源
- 手动下载模型并指定本地路径
- 设置代理服务器

在代码中可以通过捕获异常来处理下载失败的情况：

```python
try:
    bert_like, tokenizer = load_pretrained(args.bert_arch, args)
except Exception as e:
    logger.error(f"Failed to load pretrained model: {e}")
    # 处理错误或使用备用方案
```

### 跨平台差异
#### Windows系统
- 注意路径分隔符差异
- 可能需要安装 Microsoft Visual C++ Build Tools
- Docker 需要启用 WSL2 支持才能使用 GPU

#### Linux/macOS系统
- 确保有适当的文件权限
- 可能需要安装额外的系统库（如 libgl1）
- macOS 用户应注意 Apple Silicon 芯片的兼容性问题
