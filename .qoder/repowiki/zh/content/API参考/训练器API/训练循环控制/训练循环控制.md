## train_epoch 方法详解

`train_epoch` 方法实现了在一个训练周期（epoch）内执行完整的训练流程。该方法首先将模型设置为训练模式，然后遍历数据加载器中的每个批次数据。对于每个批次，系统会执行前向传播计算损失，并在存在评估指标时同时获取预测结果。在反向传播阶段，`backward_batch` 方法负责执行梯度累积、梯度裁剪、权重更新等操作。该方法支持混合精度训练（AMP），通过 `torch.amp.autocast` 上下文管理器自动处理浮点精度转换。训练过程中，系统会累积每个批次的损失值和预测结果，在一个epoch结束后计算平均损失和评估指标。当 `num_metrics` 为0时，仅返回平均损失；否则，还会返回解码器计算的评估指标。

## eval_epoch 方法详解

`eval_epoch` 方法实现了在评估模式下的完整执行流程。该方法首先将模型设置为评估模式，并在 `torch.no_grad()` 上下文中遍历开发集或测试集的数据加载器。与训练模式不同，评估过程不需要计算梯度，从而节省内存并加快计算速度。对于每个批次，系统执行前向传播获取损失和预测结果，并与真实标签进行比较。所有批次的损失值被累积后计算平均值，同时收集所有预测结果和真实标签用于最终的指标计算。该方法与学习率调度器协同工作：当 `schedule_by_step` 为 False 且调度器不是 `ReduceLROnPlateau` 类型时，学习率调度器会在每个epoch结束后调用 `scheduler.step()` 进行学习率调整。对于 `ReduceLROnPlateau` 调度器，通常在外部根据评估结果决定是否调整学习率。

## train_steps 方法详解

`train_steps` 方法实现了基于步进的训练机制，提供了更细粒度的训练控制。该方法支持定期评估、模型保存和学习率调度的集成。在训练循环中，系统会根据 `disp_every_steps` 参数定期显示运行信息，包括当前epoch、步数、学习率、损失值和评估指标。当达到 `eval_every_steps` 指定的步数时，系统会调用 `eval_epoch` 方法对开发集进行评估。评估结果用于决定是否保存最佳模型：如果 `save_by_loss` 为 True，则当开发集损失低于历史最佳值时保存模型；否则，当评估指标高于历史最佳值时保存模型。该方法还实现了与学习率调度器的协同：当 `schedule_by_step` 为 False 时，在每次评估后根据调度器类型更新学习率，对于 `ReduceLROnPlateau` 调度器，会根据损失或指标的变化决定是否降低学习率。此外，该方法支持最大步数限制和早停机制，当达到最大步数或满足早停条件时终止训练。

## 早停与多指标监控配置示例

在实际训练中，可以通过配置 `train_steps` 方法实现早停和多指标监控。例如，在文本分类任务中，可以设置 `save_callback` 回调函数来保存最佳模型，通过 `save_by_loss=False` 参数指定根据评估指标而非损失值来保存模型。当开发集的F1分数达到历史最高值时，回调函数会保存当前模型权重。为了实现早停，可以在外部监控连续多个评估周期内指标不再提升的情况，然后提前终止训练。多指标监控可以通过解码器返回的多个评估指标实现，例如同时监控精确率、召回率和F1分数。在 `text_classification.py` 脚本中，通过构建训练器并调用 `train_steps` 方法，配置了完整的训练流程，包括定期评估、模型保存和学习率调度，实现了高效的训练控制。
