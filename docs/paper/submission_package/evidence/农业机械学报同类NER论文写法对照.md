## 1. 可借鉴的共同结构

上述同类论文均不是按通用 NLP 会议论文结构组织，而是按农业信息技术论文结构展开：

1. 首页信息：中文题名、作者单位、中文摘要、关键词、中图分类号、英文题名、英文摘要和英文关键词。
2. 引言：先写作物生产和病虫害/栽培知识服务需求，再写知识图谱和 NER 的基础作用，然后综述规则、机器学习、深度学习和预训练模型，最后落到本文方法。
3. 数据集构建：单独成节，写数据来源、预处理、标注体系、实体类别和数据集划分。
4. 模型建立：按模块拆分说明，通常每个核心模块配结构图、公式和“式中…”符号解释。
5. 试验设置：写硬件/软件环境、超参数、评价指标和对比模型。
6. 结果与分析：先主模型对比，再消融或参数分析，再实体类别/泛化分析。
7. 结论：用条目式总结数据集、模型效果、关键模块贡献和应用价值。

## 2. 对当前红枣稿的启发

当前 `农业机械学报_红枣NER_投稿稿.md` 的主体框架已经接近上述三篇，但仍有三点需要按同类论文进一步打磨：

1. 数据集构建可更像茶树、水稻、苹果稿件，突出语料来源、清洗、标注、实体类别和划分依据，避免只像算法实验说明。
2. 方法章节应继续保持“模型整体结构图 -> 模块公式 -> 式中符号解释 -> 模块作用说明”的顺序。
3. 图表和 Word 版式应优先满足农业工程期刊阅读习惯：图中中文不能乱码，公式居中编号靠右，表格采用三线表，图题在图下、表题在表上。
4. 用户 DOCX 中语料来源、PDF 抽取、OCR 校对、BMES 标注和人工修正流程可吸收进 1.1 节；其中 `EDBS`、通道注意力主模型和旧数值不应直接进入当前投稿稿。

## 3. 已据此修正的排版问题

1. 5 张 SVG 图的字体栈已从 `Microsoft YaHei` 优先改为本机可用的 `Noto Sans CJK SC`，避免 Linux/LibreOffice 渲染时中文乱码或方框。
2. Word 生成脚本 `tools/md_to_docx_with_images.py` 已调整 display formula 排版：期刊格式模式下使用居中公式、右侧编号，并保留多行公式结构。
3. 主稿已吸收用户 DOCX 的材料写法，将语料来源明确为《中国果树志 枣卷》《枣丰产技术》《果树工 红枣种植》《红枣优质高效栽培技术》等，并补充 PDF 正文裁剪、OCR 校对、标点单位规范和半自动标注流程。
4. 后续重新生成 PNG、DOCX 和 PDF 后，应以 PDF 渲染结果为准继续检查图中文字和公式编号。

## 4. 后续修稿标准

若继续提高投稿质量，建议按以下顺序修：

1. 引言继续贴合“红枣栽培知识服务需求 -> 农业知识图谱 -> 农业 NER 方法进展 -> 红枣场景不足 -> 本文方法”的链条。
2. 数据集构建部分补充标注一致性或人工校验说明；若没有真实一致性统计，不应虚构数值。
3. 方法公式后增加必要的“式中…”解释，尤其是 `X`、`H^{bert}`、`D_c`、`f_{c,p}`、`s_{i,j,c}`、`p_t` 和 `γ`。
4. 结果分析避免只报数值，应解释为什么词典对农业专业术语有效、为什么边界预测改善长实体、为什么低频类别仍困难。
