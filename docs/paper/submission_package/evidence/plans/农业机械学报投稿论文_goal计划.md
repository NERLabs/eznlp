# 《农业机械学报》投稿论文 Goal 计划

## 0. Goal

以现有稿件 `docs/paper/基于词典和边界预测的红枣栽培命名实体识别.md` 为基础，完成一篇可投《农业机械学报》的完整中文论文，目标方向定位为**农业信息技术/智能农业知识工程**：围绕红枣栽培知识图谱构建中的命名实体识别问题，提出并验证基于专家词典特征与边界预测/边界选择的红枣栽培 NER 方法。

最终交付物：

1. 投稿主文稿 Markdown 版。
2. 可转 Word 的规范稿件结构。
3. 论文图表与图题表题清单。
4. 实验结果证据表和可追溯结果索引。
5. 自审意见与逐条修改记录。
6. 投稿前检查清单。

## 1. 投稿约束

期刊官网信息显示，《农业机械学报》是农业机械和农业工程类综合性学术期刊，刊登农业工程、农业信息技术、农副产品与食品科学等方向论文。投稿须知要求投稿前参照下载中心论文写作模板完善内容，参考文献至少 25 条，英文撰写参考 EI 文摘写作要求。

官网来源：

- 投稿须知：https://www.j-csam.org/jcsam/ch/first_menu.aspx?parent_id=20101116171705001
- 学报简介：https://www.j-csam.org/jcsam/ch/first_menu.aspx?parent_id=20101116171652001

对本稿的具体约束：

1. 题目需要突出农业信息技术和红枣栽培场景，不写成泛 NLP 算法论文。
2. 摘要采用“目的-方法-结果-结论”的工程论文逻辑，数值必须与正文和实验表一致。
3. 正文强调数据集构建、农业领域知识组织、模型工程价值和泛化实验。
4. 参考文献不少于 25 条，需补齐农业 NER、中文 NER、词典增强、span NER、Focal Loss、MacBERT、农业知识图谱相关文献。
5. 图表需要中英文题名，方法图、数据集统计表、对比实验表、消融实验表、泛化实验表必须齐全。

## 4. 执行计划

### Phase 1：证据归档与数值统一

目标：建立论文所有结果的唯一可信来源，避免摘要、正文、表格和结论数值不一致。

任务：

1. 汇总 RedJujube/RJND 主实验结果。
2. 汇总 baseline、BERT-wwm-ext、MacBERT、ExpertDict、Boundary Prediction、Focal Loss、完整模型结果。
3. 汇总公开数据集 MSRA、WeiboNER、ResumeNER、Boson、CLUENER 结果。
4. 建立 `paper_result_registry.md`，记录每个数值来源文件、运行时间、参数和是否采用。
5. 决定最终主结果采用 88.54 还是 88.28，必须全稿统一。

验收标准：

- 摘要中的 P/R/F1 与结果表完全一致。
- 所有“提升百分点”可以由表格直接计算。
- 每个实验数值都能追溯到本地 `results.json`、日志或分析表。

### Phase 2：期刊适配改写

目标：把稿件从“算法实验记录”改成“农业工程/农业信息技术论文”。

任务：

1. 重写摘要为目的、方法、结果、结论式表达。
2. 引言按“红枣栽培知识结构化需求 -> 农业 NER 研究现状 -> 词典增强不足 -> CRF 边界建模不足 -> 本文方法”组织。
3. 弱化与投稿方向无关的泛 NLP 细节，增强农业知识图谱、智能检索、技术问答、栽培知识抽取应用价值。
4. 将“模型创新”改写成“农业领域知识注入 + 边界预测”的问题驱动设计。
5. 补写讨论章节，明确局限性。

验收标准：

- 引言最后一段明确写出本文研究目的、方法与贡献。
- 全文不出现“代码产出”“实验尝试”等项目管理语气。
- 论文贡献不夸大，不把训练集自动抽取词典包装成人工专家知识库。

### Phase 3：方法章节规范化

目标：让方法描述和实际代码实现一致。

任务：

1. 对齐 `eznlp/model/nested_embedder.py` 中 ExpertDict 的实际实现。
2. 对齐 `eznlp/model/decoder/boundary_selection.py` 中 Boundary Selection/Boundary Prediction 的实际公式。
3. 核查是否实际使用通道注意力；如果最终实验没有采用，正文不能写成主模型组件。
4. 核查是否实际使用 BiLSTM；如果主模型为 BERT + ExpertDict + Boundary Selection，则图和公式必须一致。
5. 明确 Focal Loss、边界平滑、span size embedding、最大 span 长度等参数。

验收标准：

- 方法图中每个模块都能在代码中找到对应实现。
- 公式符号前后一致。
- 模型名、缩写、解码器名称全稿一致。

### Phase 4：图表重构

目标：生成适合《农业机械学报》的论文图表，不使用 Mermaid 作为最终投稿图。

必备图：

1. 图 1：EDBP 总体结构图。
2. 图 2：专家词典 BMES 四通道编码示意图。
3. 图 3：边界预测解码器 span 分类示意图。
4. 图 4：不同实体类别识别 F1 对比图。
5. 图 5：长实体识别/边界错误案例分析图。

必备表：

1. 表 1：RJND 实体类别及数据集分布。
2. 表 2：实验参数设置。
3. 表 3：主模型与基线模型对比。
4. 表 4：消融实验。
5. 表 5：词典策略对比。
6. 表 6：公开数据集泛化实验。
7. 表 7：不同实体类别 P/R/F1。

验收标准：

- 图题、表题中英文对应。
- 表格数值保留位数统一。
- 图不依赖代码块渲染，后续可导出 PNG/TIFF 或矢量图。

### Phase 5：内部审稿

目标：模拟农业信息技术方向审稿，提前处理硬伤。

审稿维度：

1. 选题是否符合《农业机械学报》农业信息技术方向。
2. 数据集是否有充分农业领域说明。
3. 方法创新是否清楚，是否只是已有 NER 模块堆叠。
4. 实验对比是否公平。
5. 消融是否能支持每个创新点。
6. 泛化实验是否服务主结论，而不是分散主题。
7. 参考文献是否覆盖近 5 年农业 NER 和中文 NER。
8. 是否存在数据泄漏风险，尤其是从训练集自动抽取词典的描述。

预期主要审稿意见与应对：

1. **农业工程属性不足**：增加红枣栽培知识图谱、农业技术文本结构化和智能检索应用讨论。
2. **方法创新不足**：突出“训练集实体自动词典 + BMES 四通道位置先验 + 边界预测”组合针对农业术语边界识别的作用，并用消融证明。
3. **实验结果偏低或与旧稿不一致**：统一最终主结果，解释数据集难度和类别不平衡。
4. **公开数据集过多分散**：将其定位为泛化验证，篇幅控制在一节。
5. **术语混乱**：统一 EDBP、Boundary Prediction、Boundary Selection 的关系。

验收标准：

- 输出一份 `internal_review_round1.md`。
- 每条审稿意见都有对应修改动作。
- 修改后输出 `revision_log_round1.md`。

### Phase 6：参考文献补齐

目标：满足投稿至少 25 条参考文献要求，并提高近年文献比例。

文献类型配比：

1. 农业 NER/农业知识图谱：8-10 条。
2. 中文 NER 与预训练模型：5-7 条。
3. 词典增强 NER：4-5 条。
4. Span NER/Boundary Selection/Biaffine：4-5 条。
5. Focal Loss/类别不平衡：2-3 条。
6. 红枣栽培或农业技术知识组织背景：2-4 条。

验收标准：

- 参考文献不少于 25 条。
- 格式按 GB/T 7714 风格整理。
- 正文引用顺序与文末编号一致。
- MacBERT、BERT、BiLSTM-CRF、SoftLexicon、Lattice LSTM、biaffine/span NER、Focal Loss 均有正式引用。

### Phase 7：投稿前终检

目标：形成可提交版本。

检查项：

1. 题名不宜过长，突出红枣栽培和方法核心。
2. 中文摘要、英文摘要、关键词对应。
3. 英文摘要满足 EI 摘要逻辑，包含目的、方法、结果、结论。
4. 公式编号连续，符号解释完整。
5. 图表编号连续，正文均有引用。
6. 结果数值全稿一致。
7. 结论只总结正文已验证内容，不引入新结果。
8. 参考文献不少于 25 条。
9. 不出现 Markdown 代码块、Mermaid 源码等非投稿内容。
10. 数据集、代码、模型可复现实验说明清楚。
